# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Educational cross-platform 3D engine. Linux/Vulkan, macOS/Metal, Windows/DX12. Not production.

## Build

After clone: `git submodule update --init`

Get SLANGC from https://github.com/shader-slang/slang/releases → `scripts/tools/slang/`
Windows also needs DXC from https://github.com/microsoft/DirectXShaderCompiler/releases → `scripts/tools/dxc/`

| Platform | Command |
|----------|---------|
| Linux | `./scripts/build_linux.sh` |
| macOS | `./scripts/build_mac.sh` |
| Windows | `scripts\build_win.bat` |

Output: `bin/app` (or `bin/app.exe`), `bin/shaders/`

## Tests

Single test (5-tick headless run):
```bash
./bin/app --script tests/cube_gpu_test.as --ticks 5
```

All tests:
```bash
./scripts/run_tests.sh          # Linux/Mac
scripts\run_tests.bat           # Windows
```

Logs → `scripts/logs/`

## Architecture

```
engine (main loop)
├── script_manager         AngelScript load + execute
├── window_manager         GLFW windows
├── renderer               frame management, GPU command buffer fill
│   └── dynamic_rhi        abstract RHI interface
│       ├── vk_dynamic_rhi Vulkan (Linux)
│       ├── m_dynamic_rhi  Metal (macOS)  — .mm files, OBJC/OBJCXX
│       └── dx_dynamic_rhi DirectX 12 (Windows)
├── scene                  render_objects + camera
└── texture_manager        async load + cache
```

**Modules** (each a static lib linked into main app):
- `lucus-sim` — engine, script integration
- `lucus-core` — base classes, scripting bindings, filesystem
- `lucus-graphics` — RHI, scene, material, mesh, texture
- `lucus-math` — transform, camera (wraps GLM)

## RHI Design

Abstract base `dynamic_rhi` with virtual methods. Factory `create_dynamic_rhi()` returns platform impl.

**Handle pattern** — GPU resources are opaque handles (`mesh_handle`, `texture_handle`, `pipeline_state_handle`, `uniform_buffer_handle`, etc.) generated by `CUSTOM_RESOURCE_HANDLE(T)` macro. Handle stores u32 index (1-indexed; 0 = invalid). RHI impls store actual resources in hash maps keyed by handle index.

**GPU command buffer** — renderer records fixed-size command structs (≤32 bytes) into byte buffer; RHI `execute()` consumes it. Commands: `RENDER_PASS_BEGIN/END`, `BIND_PIPELINE`, `BIND_TEXTURE`, `BIND_VERTEX_BUFFER`, `DRAW_INDEXED`, etc.

## Key Patterns

**Intrusive ref-count**: `object` base class + `intrusive_ptr<T>` smart pointer. Used for mesh, material, texture, camera, scene, render_object.

**Singleton**: template base `singleton<T>` → `window_manager`, `renderer`, `script_manager`, `texture_manager`.

**Factory methods**: `material::create_factory()`, `mesh::create_factory()`, `texture::create_factory()`.

**Script bindings**: `bind_*.cpp` files expose C++ types to AngelScript. Game/test logic lives in `content/scripts/*.as`.

## Shaders

Language: SLANG. Sources in `shaders/*.slang`.
Compiled to `bin/shaders/` as `*.spv` (Vulkan), `*.dxil` (DX12), `*.metallib` (Metal).

## Third-Party

Git submodules: GLFW 3, AngelScript, GLM.
Included directly: TinyGLTF v3, DirectX SDK (Windows, `third-party/directx/`).
